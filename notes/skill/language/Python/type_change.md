## 字符串与整型的本质差异

之前对“字符串转整数”这件事一直模糊，现在理清一下：

- **字符串（`str`）**：本质是按顺序排列的字节序列。`"100"` 由字符 `1`、`0`、`0` 组成，每个字符占 1 字节（若使用 ASCII/UTF-8），所以一共是 3 字节。
- **整型（`int`）**：本质是数值。Python 的 `int` 没有固定字节数，但在底层会按需要使用若干个字节去表示它的数值，比如 C 语言里的 32 位 `int` 会用 4 字节存放 `100` 的二进制编码。

由此可见，字符串和整型的“码制”完全不同：一个是“字节里的字符”，另一个是“字节里的数值”。

## 转换时发生了什么

当执行 `int("100")` 时：

1. 解释器先逐个读取字符 `1`、`0`、`0`；
2. 按十进制解析出真正的数值 100；
3. 再把 100 存入整型对象，占用的字节数取决于实现（Python 会动态分配足够的空间）。

所以，转换前后相同的“可读形式”并不代表底层二进制一样：`"100"` 的三个字节被重新解释成了一个整数值 100。理解这一点即可，不必纠结内部的每一步细节，看到输入输出就能判断结果。


cpu不需要区别这些东西，因为这些都是由编译器和解释器处理的。对于 CPU 来说，它只处理二进制数据，至于这些数据代表什么（字符串还是整数）是由程序员和编译器定义的。CPU 执行指令时会根据指令类型来解释这些数据，但它并不关心数据的“类型”概念，这些都是在更高层次的抽象中处理的。
