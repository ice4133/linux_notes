Python代码执行
    ↓
from unitree_actuator_sdk import *
    ↓
Python查找 PYTHONPATH 中的 unitree_actuator_sdk（.so文件）
    ↓
找到 .cpython-38-x86_64-linux-gnu.so
    ↓
系统加载这个.so文件
    ↓
.so文件需要依赖 libUnitreeMotorSDK_Linux64.so
    ↓
系统查找 LD_LIBRARY_PATH 来找依赖的库
    ↓
成功加载所有.so文件
    ↓
Python代码可以调用 SerialPort、MotorCmd 等



两个办法：
一个是分别配置python和.so

一个是直接配置setup.bash

```bash
# 1. 将 sdk_lib 路径加入 Python 搜索路径
export PYTHONPATH=$PYTHONPATH:~/my_dog/dog_ws/src/resource/robot/sdk_lib

# 2. 将 sdk_lib 路径加入系统库搜索路径 (防止报链接错误)
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:~/my_dog/dog_ws/src/resource/robot/sdk_lib

# 3. 再次运行脚本
python3.8 motor_init.py


# 这样就解决了

```
