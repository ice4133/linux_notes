# 接口

std_msgs::msg::String msg;
本质上就像在写 std::string s;

只不过我们日常应用都是using namespace std;
std::和std_msgs::同样都是一种
C++ 标准库将所有的函数、类、对象都封装在 std 命名空间中。

std:: 这个原来是把所有东西都封装在这里面，很像库
命名空间是不是很像库？
回答：是的，命名空间确实很像库。它们都是为了组织代码和避免命名冲突而设计的。命名空间可以看作是一个容器，用于存放相关的函数、类和对象，就像一个库一样。通过使用命名空间，我们可以更好地管理代码，并且在不同的库或模块中使用相同的名称而不会发生冲突。


**类比理解：**
- 接口 ≈ C++中的数据类型（如 std::string）
- 命名空间 ≈ 库的分类容器



# 类比理解
ros解决的问题是不同的进程之间如何通信
就像不同的程序之间如何通信一样

它发明了四种通信方式：话题、服务、动作、参数服务器

按理来说，就好像 struct data 一样，数据结构的定义和数据内容是要放在一起的

但是因为有python和c++两种语言，为了让两种语言都能使用，所以就把数据结构的定义放在一个单独的文件里，叫做接口文件（.msg文件）

对于话题来说，接口文件是定义话题消息的数据结构的文件
struct data 
接口 话题


对于服务来说，接口文件是服务的函数声明，必须按照这个来
int func(int a) //接口
int func(int a)
{
    //服务
}



对于动作来说，接口文件就是任务说明书，可以不那么完整，但是大纲必须要有


也就是说内容是通信机制
