# 寻址方式（针对操作数）
+ 立即寻址：操作数直接在指令中给出
+ 寄存器寻址：操作数在寄存器中
+ 存储器寻址：操作数在内存中，通过地址访问
+ I/O 寻址：操作数在输入输出设备中，通过端口访问

I/O寻址分为两种：
+ 端口寻址：通过特定的端口号访问 I/O 设备
假如提供8位地址总线和8位数据总线，CPU可以访问256个端口，每个端口可以存储一个字节的数据。
例如有一个指令 `IN AL, 0x10`，表示从端口号为0x10的I/O设备读取一个字节数据到AL寄存器中。

+ 内存映射寻址：将 I/O 设备的寄存器映射到内存地址空间，通过内存地址访问 I/O 设备


存储器寻址分为多种：
+ 直接寻址：指令中给出操作数的内存地址
给变量也算直接寻址，例如指令`MOV AX, BUFFER`，其中 BUFFER 是一个变量，编译器会将其地址直接放在指令中。

+ 寄存器间接寻址：指令中给出一个寄存器，寄存器中存储了另一个寄存器的地址，最终通过这个寄存器访问内存
例如指令 `MOV AX, [BX]`，表示将内存地址由寄存器 BX 指向的内容加载到 AX 寄存器中。

+ 基址寻址：指令中给出一个基址寄存器和一个偏移量，通过基址寄存器的值加上偏移量来计算操作数的内存地址
+ 变址寻址：指令中给出一个变址寄存器和一个偏移量，通过变址寄存器的值加上偏移量来计算操作数的内存地址    
+ 基址变址寻址：指令中给出一个基址寄存器、一个变址寄存器和一个偏移量，通过基址寄存器的值加上变址寄存器的值再加上偏移量来计算操作数的内存地址
例如指令 `MOV AX, [BX + SI + 4]`

# 指令（针对操作码）

## 数据传送指令

### 通用传送指令：MOV
格式：`MOV 目的操作数, 源操作数`
功能：将源操作数的值传送到目的操作数中

注意：
+ 目的操作数不能是立即数，

+ 源操作数可以是寄存器、内存地址或立即数，但不能同时为内存地址
如果一条指令要求“从内存 A 读”同时“向内存 B 写”，这意味着在同一个指令周期内要进行两次内存访问。
由于内存访问相对较慢，CPU无法同时完成两次内存访问，因此这样的指令是非法的。例如，`MOV [1000h], [2000h]` 是非法的，因为它试图从内存地址 2000h 读取数据并直接写入内存地址 1000h，这需要两次内存访问，CPU无法在一个指令周期内完成。


+ 不能同时为段寄存器
段寄存器不同于普通寄存器，它们专门用于存储内存段的基地址，因此在数据传送指令中不能同时使用两个段寄存器作为操作数。例如，`MOV DS, ES` 是非法的，因为它试图将一个段寄存器的值传送到另一个段寄存器中，这种操作没有实际意义且不被允许。

+ 在源操作数是立即数时，目的操作数不能是段寄存器，例如 `MOV AX, 1234h` 是合法的，但 `MOV DS, 1234h` 是非法的。

+ IP和CS寄存器不能作为目的操作数，例如 `MOV IP, AX` 和 `MOV CS, AX` 都是非法的。

例如：`MOV AX, BX`（寄存器到寄存器），`MOV AX, [1000h]`（内存到寄存器），`MOV [1000h], AX`（寄存器到内存），`MOV AX, 1234h`（立即数到寄存器） 

不可以是因为没有对应的机器码，CPU无法识别这样的指令。
