# 第二步
其实主要还是如何在仓库里面添加各种各样的一些小细节，让自己开发的更加舒服

## 1.添加一个README.md文件
这个文件主要是用来介绍这个仓库的，主要是用来介绍这个仓库的功能


## 2.添加一个.gitignore文件
这个文件主要是用来告诉git哪些文件不需要被提交到仓库里面去

语法

直接写名称，它会直接忽略同名的文件，比如build，只要匹配到build这个名字的文件或者文件夹都会被忽略掉

/  表示锁定当前.gitignore文件所在的目录

/结尾，比如tmp/，表示锁定当前.gitignore文件所在的目录下的tmp这个文件夹，这个文件夹下面的所有文件都会被忽略掉
