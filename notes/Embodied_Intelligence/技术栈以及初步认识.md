# 技术栈
## 第一阶段：工具与语言准备（打地基）
MuJoCo 不是一个独立的“软件”（像 Word 那样），它是一个函数库。你得先学会怎么跟计算机说话。Python 基础： 你不需要成为高手，但必须掌握：变量、列表（List）、循环（For loop）、函数（Function）以及**面向对象编程（Class）**的基础概念。
NumPy 库： 这是重中之重。MuJoCo 所有的位姿、速度、力矩数据都是以 矩阵（Matrix） 和 向量（Vector） 形式存在的。如果你看不懂 data.qpos[0:7] 是什么意思，后面会非常痛苦。VS Code 操作： 学会配置虚拟环境，解决“找不到包”的问题。

## 第二阶段：MJCF 语法（理解“机器人的身体”）
在写 Python 代码之前，你必须先学会如何用 XML 描述一个机器人。MuJoCo 的原生语言叫 MJCF。理解层级结构： 知道什么是 worldbody（世界），什么是 body（零件），零件之间是怎么嵌套的。掌握核心标签：<joint>：关节（它是怎么转的？是滑动的还是球形的？）。<geom>：形状（它是球体、胶囊还是你导入的 STL 网格？）。<site>：位置点（传感器挂哪？力往哪施加？）。坐标系概念： 搞清楚相对坐标（父物体）和绝对坐标（世界）。

## 第三阶段：机器人学基础概念（补齐“消失的背景知识”）
为什么教程里总提到 qpos、qvel、ctrl？这些是机器人学的通用术语。广义坐标 (Generalized Coordinates)： 简单说，就是用最少的数字描述机器人的姿态。自由度 (DOF)： 你的机器人能怎么动？运动学 (Kinematics)： 知道角度，求手爪的位置。动力学 (Dynamics)： 知道给多少力，求机器人会怎么加速。姿态表示： 至少要知道什么是 四元数 (Quaternion)，因为 MuJoCo 默认不用欧拉角（角度/弧度），这可能是你最看不懂的地方。

## 第四阶段：MuJoCo Python API（写“大脑”）
当你能看懂 XML，也知道什么是矩阵了，再去翻官方文档或看视频。加载与仿真： 学会 mj_step，让物理时间流动。数据读取： 学会从 mjData 中提取机器人的状态。控制输入： 学会把计算出的力矩填进 data.ctrl。交互： 学会用 mujoco.viewer 实时观察。
