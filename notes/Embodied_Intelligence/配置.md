# 下载软件

+ 到github上面下载mujoco，并且sudo dpkg -i安装

+ 配置环境变量
打开终端
vim ~/.bashrc

```
# MuJoCo 配置
export MUJOCO_PATH="/home/ice/Apps/mujoco-3.5.0-linux-x86_64/mujoco-3.5.0"
export PATH="$MUJOCO_PATH/bin:$PATH"
export LD_LIBRARY_PATH="$MUJOCO_PATH/bin:$LD_LIBRARY_PATH"
```

+ 打开并使用

找到与bin文件夹同级的model/humanoid/humanoid.xml

```
simulate /home/ice/Apps/mujoco-3.5.0-linux-x86_64/mujoco-3.5.0/model/humanoid/humanoid.xml
```


# python环境配置

如果只用simulate ，那么就只是调用，观察，想要真正实现一个任意情况的仿真，需要配置python环境

一般来说ubuntu是自带python环境的，可是为什么还是要下载呢？

ubuntu自带python3 ，但是linux中的默认指令是python2 ， 环境不兼容，所以仍然需要配置环境

也就是让ubuntu知道 它默认的python指令调用的是python3


+ 下载pyhton3 和 pip
```
sudo apt install pip

sudo apt install python-is-python3

```

+ 测试
在只执行了sudo apt install pip后

先执行
```
python --verison

python3 --version
```

在执行完sudo apt install python-is-python3后

再执行
```
python --verison

python3 --version
```

+ 配置vscode
如果有需要，最好配置一个专门的python工作区


# Mujoco配置总结

对于一个有python环境的vscode ， 配置Mujoco有两步，第一步下载Mujoco，第二步是pip install mujoco
MuJoCo 安装包只提供底层引擎，需要你在对应语言里配置好库路径并引入合适的绑定；而 pip install mujoco 已经把这两部分打包好了。如果你想“用自己的引擎”，就显式设好路径，让绑定去加载它。


第二步可以用其它语言，但用pip，那就是默认用python
