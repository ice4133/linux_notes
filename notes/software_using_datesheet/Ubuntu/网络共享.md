# Unitree G1 MiniPC 网络配置说明

## 1. 问题背景

在使用宇树（Unitree）G1 人形机器人时，经常会通过 SSH 登录机器人内部的 miniPC：

```bash
ssh 用户名@机器人IP
```

SSH 可以连接说明：

- 电脑和 G1 miniPC 之间的局域网通信正常；
- 网线连接正常；
- IP 地址配置正确。

但是：

```bash
sudo apt update
git clone xxx
pip install xxx
ping github.com
```

可能失败。

原因是：

> G1 miniPC 默认网络通常只用于机器人内部通信，没有连接互联网的默认网关。

也就是说：

```
电脑  <------网线------>  G1 miniPC

可以通信

但是：

G1 miniPC ----X---- Internet
```

G1 不知道如何访问外部网络。

解决方法：

让电脑充当一个“小路由器”，把自己的 WiFi 网络共享给 G1。

---

# 2. 网络结构理解

配置完成后的网络结构：

```
                 Internet
                    |
                    |
                  WiFi
                    |
             +--------------+
             |   我的电脑    |
             |              |
             | WiFi 网卡     |
             | 网口          |
             +--------------+
                    |
                  网线
                    |
             +--------------+
             | G1 miniPC    |
             +--------------+
```

数据路径：

```
G1 miniPC
    |
    |
电脑网口
    |
    |
电脑WiFi
    |
    |
Internet
```

电脑负责：

1. 接收 G1 发出的网络请求；
2. 修改数据包地址（NAT）；
3. 从 WiFi 发出去。


---

# 3. 第一步：查看电脑网络接口

在自己的 Ubuntu 电脑执行：

```bash
ip a
```

例如：

```
2: wlp3s0:
    inet 192.168.1.50

3: enp4s0:
    inet 192.168.123.100
```

含义：

|网卡|作用|
|-|-|
|wlp3s0|无线 WiFi，连接互联网|
|enp4s0|有线网口，连接 G1|

不同电脑名字可能不同：

无线：

```
wlan0
wlp2s0
wlp3s0
```

有线：

```
eth0
enp3s0
enxxxxx
```

需要确认：

- 哪个接口能上网；
- 哪个接口连接机器人。

---

# 4. 第二步：电脑开启网络转发

Linux 默认禁止数据包从一个网卡转发到另一个网卡。

开启：

```bash
sudo sysctl -w net.ipv4.ip_forward=1
```

查看：

```bash
cat /proc/sys/net/ipv4/ip_forward
```

如果输出：

```
1
```

表示开启成功。

---

# 5. 第三步：电脑配置 NAT 转发

假设：

WiFi：

```
wlp3s0
```

连接 G1：

```
enp4s0
```

执行：

```bash
sudo iptables -t nat -A POSTROUTING \
-o wlp3s0 \
-j MASQUERADE
```

作用：

让 G1 发出的数据经过电脑 WiFi 出口访问互联网。


允许数据转发：

```bash
sudo iptables -A FORWARD \
-i enp4s0 \
-o wlp3s0 \
-j ACCEPT
```

允许返回数据：

```bash
sudo iptables -A FORWARD \
-i wlp3s0 \
-o enp4s0 \
-m state --state RELATED,ESTABLISHED \
-j ACCEPT
```

电脑端配置完成。

# 7. 查看 G1 网络信息

执行：

```bash
ip a
```

例如：

```
eth0:
    inet 192.168.123.164
```

说明：

G1 地址：

```
192.168.123.164
```

再查看路由：

```bash
ip route
```

可能看到：

```
192.168.123.0/24 dev eth0
```

但是没有：

```
default via xxx
```

这说明：

> G1 知道如何访问电脑，但是不知道互联网出口在哪里。

---

# 8. 给 G1 设置默认网关

假设：

电脑连接 G1 的网口 IP：

```
192.168.123.100
```

那么 G1 执行：

```bash
sudo ip route add default via 192.168.123.100
```

含义：

以后访问外网：

```
G1
 |
电脑
 |
Internet
```

---

# 9. 设置 DNS

即使 IP 可以访问，如果 DNS 不正常：

```
ping 8.8.8.8
```

成功：

但是：

```
ping github.com
```

失败。

查看 DNS：

```bash
cat /etc/resolv.conf
```

如果没有 nameserver：

编辑：

```bash
sudo nano /etc/resolv.conf
```

添加：

```
nameserver 8.8.8.8
```

保存。

# 11. 常见问题排查


## 问题1：SSH正常，但是 ping 失败

检查：

```bash
ip route
```

是否存在：

```
default via xxx
```

如果没有：

添加：

```bash
sudo ip route add default via 电脑IP
```


---

## 问题2：ping 8.8.8.8成功，但是ping域名失败

DNS问题。

检查：

```bash
cat /etc/resolv.conf
```

添加：

```
nameserver 8.8.8.8
```


---

## 问题3：G1能访问电脑，但是不能访问互联网

检查电脑：

开启转发：

```bash
cat /proc/sys/net/ipv4/ip_forward
```

应该：

```
1
```

检查 NAT：

```bash
sudo iptables -t nat -L
```

应该存在 MASQUERADE。

# 12. 推荐开发流程

对于机器人开发，推荐：

```
电脑
 |
 | SSH
 |
G1 miniPC
 |
 |
开发环境配置
 |
 |
ROS2 / SDK / Python / C++
```

电脑负责：

- 网络共享；
- VS Code Remote SSH；
- 文件传输；
- 编译调试。

G1 miniPC负责：

- 运行机器人程序；
- 控制算法；
- ROS2节点；
- 推理模型。

# 13. 最重要的检查顺序

不要一开始修改配置。

按照顺序：

## 电脑：

```bash
ip a
```

确认：

- WiFi接口；
- G1网口接口。


## G1：

```bash
ip a
ip route
```

确认：

- G1 IP；
- 是否缺少 default route。


## 测试：

```bash
ping 8.8.8.8
```

然后：

```bash
ping github.com
```


只有确认网络结构后，再修改路由。

---

# 总结

G1 miniPC 无法联网，本质不是 SSH 问题，而是：

> G1 只有局域网连接，没有互联网出口。

解决方案：

1. 电脑连接 WiFi；
2. 电脑网口连接 G1；
3. 电脑开启 IP 转发；
4. 电脑配置 NAT；
5. G1 设置电脑为默认网关；
6. 配置 DNS。

完成后：

G1 miniPC 就可以：

- apt安装软件；
- git下载代码；
- pip安装Python包；
- 配置ROS2环境。
