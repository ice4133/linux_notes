# vnc或者nomachine或者ssh+todesk
这个的目的是能操控并且能够看到minipc显示器的界面


vnc
在小电脑里面设置即可

无法全屏使用
minipc使用1920*1080的显示器，
电脑使用2560*1600

在小电脑终端输入
export DISPLAY=:0



切换缩放模式


你说
如果在使用vnc,那么画面的流畅度，是不是跟网络有关系，是不是用网卡的速度会快很多？如果我用vnc，那么如果跑ros2用了很多性能的话，那么是不是会很卡

nomachine效果比vnc好很多，画面更流畅，支持全屏
1. 硬件加持（最直接）： 用一根网线直接连接 ThinkPad 和 MiniPC。给两块网卡设置固定 IP（比如 192.168.1.5 和 192.168.1.6）。这样带宽不再是瓶颈
2. 降低 VNC 的质量（保流畅）：
在 Remmina 的设置里，把“质量”调低。虽然画面会有马赛克，但响应速度会变快。
3. 避开“视频流”：
不要在 VNC 窗口里看高频变化的画面（比如摄像头的实时预览图）。你可以用终端看日志，图像显示交给专门的轻量化工具。
4. 改用 NoMachine（逻辑优化）：
我还是要多提一句，NoMachine 之所以比 VNC 快，是因为它有硬件加速抽吸技术。它能利用 MiniPC 的核显来压缩画面，不怎么占 CPU

# 总结

ssh可以适合只用终端的情况，vnc和nomachine适合需要图形界面的情况，尤其是当需要运行一些图形化工具（比如Rviz2）的时候，nomachine的表现会更好一些。
但是用摄像头的话，ssh就不行了，根本看不到


此时vnc有优势，只需要export DISPLAY=:0 就可以了，虽然画面可能不太流畅，但是至少能看到摄像头的实时预览图了。
