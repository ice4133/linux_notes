# 类比

|维度|手机操作系统 (如 Android)|机器人操作系统 (ROS)|
|---|---|---|
|底层内核|基于 Linux 内核|基于 Linux 内核|
|硬件抽象|摄像头、GPS、加速度计的驱动|雷达、IMU、电机驱动、机械臂接口|
|软件单元|应用程序（App）|节点（Node）|
|通信机制|Intent、Broadcast Receiver|Publisher-Subscriber、Service、Action|
|功能扩展|应用商店|GitHub、ROS Index|
|用户界面|图形用户界面（GUI）|命令行界面（CLI）、RViz、rqt|
|开发语言|Java、Kotlin、C++|Python、C++|

使用sudo apt install 下载的ROS软件包，会放在/opt/ros/humble目录下，而使用源码安装的ROS软件包则会放在用户指定的目录下，通常是~/ros2_ws/src目录下。

# 一些指令

查出apt下载的功能包
```bash
ros2 pkg list #列出所有安装的ROS包
ros2 pkg prefix <package_name> #显示指定包的安装路径

ros2 pkg executables <package_name> #列出指定包中可执行文件的路径
ros2 pkg executables #列出所有包中可执行文件的路径
ros2 pkg xml <package_name> #显示指定包的XML文件内容
```


# 分类介绍

通过 apt 下载的程序不需要使用 colcon，只有你自己编写的或从 GitHub 等平台下载的源代码才需要 colcon

通常来说可以不使用colcon，但是如果你想要编译自己的ROS包，或者从GitHub上下载的ROS包没有预编译版本，那么你就需要使用colcon来构建这些包。colcon是ROS 2的构建工具，可以帮助你编译和安装ROS包，使它们能够在你的系统上运行。

ros2默认不会下载colcon，所以如果你需要使用colcon来构建ROS包，你需要手动安装它。你可以使用以下命令来安装colcon：

```bash
sudo apt install python3-colcon-common-extensions


# 检验
colcon --help
```
