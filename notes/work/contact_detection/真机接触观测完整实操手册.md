# G1 真机接触观测完整实操手册

这是一份以命令和验收结果为主的现场手册。背景原理、安装细节和故障解释见
`真机算法测试方案.md`；现场操作优先按照本文执行。

## 0. 先看当前代码能做什么

当前代码已经具备：

```text
[可执行] 读取 LowState、主 IMU、torso IMU 和 LowCmd
[可执行] 生成 50 Hz 同步快照
[可执行] fixed-base / floating-base Pinocchio 计算
[可执行] 左右腕六维 wrench 计算
[可执行] 无接触 PASS/FAIL 和 valid_ratio/pass_ratio 统计
```

当前代码尚未具备：

```text
[未实现] 50 Hz 全量 CSV 原始数据记录器
[未实现] 自动计算基线均值、标准差和分位数
[未实现] 基线标定文件生成和加载
[未实现] 在关节残差中自动减去基线
[未实现] 完整 FloatingBaseKinematics 实时数据源
[未实现] 将完整基座状态与 LowState 因果同步
```

因此当前最多可以完成：

```text
L1：真机只读链路配置完成
G4：当前静态姿态的无接触质量验收
```

没有经过验证的姿态控制器时：

```text
G5 = NOT READY
不得临时编写 LowCmd 发布器
不得启动未经验证的 SONIC 只为改变姿态
不得取消吊架
```

### 0.1 四个容易混淆的名称

它们不是“测出四个常数，再一起填进Pinocchio”：

| 名称 | 实际是什么 | 是否测一次后保存 | 当前状态 |
| --- | --- | --- | --- |
| 静态基座 | 机器人静止时采用 `--fixed-base` 假设 | 否，只是计算模式 | G3B～G7可用 |
| 动态基座 | 每一帧的位置、姿态、速度和加速度 | 否，必须50 Hz实时输入 | 尚未接入 |
| 静态基线 | 无接触静止时的7维关节残差偏置及波动 | 是，应标定和保存 | 当前只检查，未生成文件 |
| 动态误差模型 | 速度、加速度、温度相关的摩擦和模型误差 | 需要受控运动辨识 | G5以前不能做 |

实际数据链路是：

```text
29关节状态 + 基座实时状态
        -> Pinocchio Compute(...)
        -> 左右臂各7维原始关节残差
        -> 减静态基线/动态误差
        -> Estimate(...)
        -> 左右腕各6维力/力矩
```

所以：

```text
G3B使用静态基座假设验证动力学
G4只验证当前姿态零载结果是否合格
G5采集多个静态姿态，才有资格建立静态基线
受控动态运动和完整基座状态具备后，才辨识动态误差模型
```

## 1. 完成等级

| 等级 | 必须完成 | 允许给出的结论 |
| --- | --- | --- |
| L1 | G0、G1、G2、G3A、G3B | DDS、同步、URDF和静态动力学链路正常 |
| L2 | L1、G4、G5、G6、G7、正式基线和阈值配置 | 静态接触观测配置完成 |
| L3 | L2、完整基座状态、回放、吊架动态、落地动态 | 动态浮动基座配置完成，可申请接入RL observation |

四个 reader 各运行一次不等于 L2 或 L3 完成。

# 第五部分：以后获得姿态控制器后的G5～G7

## 17. G5：多静态姿态

每个姿态都按同一流程：

```text
1. 吊架保持
2. 退出reader
3. 由已验证控制器缓慢移动到姿态
4. 停止运动
5. 等待至少3秒
6. 确认手臂无接触、吊架不晃
7. 启动fixed-base采集
8. 检查PASS
9. 每个姿态重复至少3次
```

建议姿态：

```text
neutral
left_arm_forward
right_arm_forward
both_arms_forward
both_arms_side
left_elbow_bent
right_elbow_bent
different_wrist_orientation
```

每个姿态命令：

```bash
export POSE_NAME=left_arm_forward

./target/release/contact_wrench_reader \
  "$ROBOT_IF" \
  --fixed-base \
  --expect-no-contact \
  --warmup-seconds 3 \
  --duration-seconds 20 \
  --max-no-contact-force 15 \
  --max-no-contact-moment 3 \
  --minimum-pass-ratio 0.95 \
  2>&1 | tee "$TEST_SESSION/05_${POSE_NAME}.log"
```

通过条件：

```text
overall_result=PASS
valid_ratio >= 0.99
pass_ratio >= 0.95
rank=6
saturated=no
```

正式采集期间机器人必须静止。G5不是动态测试。

只有G5通过后才能执行。

同一姿态先测无负载：

./target/release/contact_wrench_reader \
  "$ROBOT_IF" \
  --fixed-base \
  --warmup-seconds 5 \
  --duration-seconds 30 \

  --fixed-base \
  --warmup-seconds 5 \
  --duration-seconds 30 \

左右腕分别测试：

## 21. 当前基线有没有自动输入算法

没有。

当前运行路径是：

```cpp
auto dynamics_result = dynamics.Compute(snapshot);
auto wrench_result = estimator.Estimate(dynamics_result);
```

它直接使用原始关节残差，没有加载或减去基线。

正式路径应开发为：

```cpp
auto dynamics_result =
    dynamics.Compute(snapshot, base_kinematics);

g1::contact_observation::types::ArmVector left_external{};
g1::contact_observation::types::ArmVector right_external{};

for (std::size_t i = 0;
     i < g1::contact_observation::types::kArmMotorCount;
     ++i) {
  left_external[i] =
      dynamics_result.left_arm_tau_residual_raw_nm[i] - left_baseline[i];
  right_external[i] =
      dynamics_result.right_arm_tau_residual_raw_nm[i] - right_baseline[i];
}

auto wrench_result =
    estimator.Estimate(
        dynamics_result,
        left_external,
        right_external);
```

`Estimate(dynamics, left, right)`重载已经存在；基线记录、生成和加载尚未实现。

# 第七部分：基座数据的实操状态

## 23. 当前完整基座数据保存在哪里

当前不存在完整基座数据源，也没有基座数据文件。

当前只有：

```text
snapshot.main_imu.quaternion
snapshot.main_imu.gyroscope
```

它们保存在内存环形缓冲区，程序退出后消失。

当前reader调用：

```cpp
Compute(snapshot);
```

因此得到：

```text
base_kinematics_complete=false
residual_reliable=no
```

## 24. 完整基座以后怎么输入

必须先找到或开发可靠状态估计器，每个50 Hz周期提供：

```cpp
FloatingBaseKinematics base;
base.position_world_m = ...;
base.orientation_wxyz = ...;
base.linear_velocity_world_mps = ...;
base.angular_velocity_base_radps = ...;
base.linear_acceleration_world_mps2 = ...;
base.angular_acceleration_base_radps2 = ...;
```

然后与LowState按时间戳对齐，再调用：

```cpp
auto dynamics_result =
    dynamics.Compute(snapshot, base);
```

基座状态是每帧实时输入，不是一个固定参数文件。

原始IMU加速度不能直接填入 `linear_acceleration_world_mps2`。状态估计器必须
