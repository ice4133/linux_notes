# 第一步，配置雷达的SDK
雷达的SDK本质上是读取雷达的一些二进制数据，给用户提供一个接口，用户调用这个接口就可以得到雷达的数据了。

这也是对于所有这种类型的设备的SDK的通用设计，通过SDK，得到一个接口，用户调用这个接口就可以得到设备的数据了。

```bash
# 配置编译环境
sudo apt update
sudo apt install cmake pkg-config -y


# 下载并且编译SDK
git clone https://github.com/Livox-SDK/Livox-SDK2.git
cd Livox-SDK2
mkdir build && cd build
cmake .. && make -j$(nproc)
sudo make install


# sudo make install 是把编译好的库文件和头文件那些的安装到minipc下面，这一步至关重要

# 查询是否配置好,这个是查询是否有头文件和库文件，有就成功了
# 理论上来说，一个闭源的项目，只需要头文件和由各种源文件编译好的库文件就可以了，也就是到现在，得到了一个系统环境，可以调用雷达的api了
ls /usr/local/lib | grep livox
ls /usr/local/include | grep livox


```



# 第二步，配置雷达节点

因为基于ROS2开发，所以需要配置一个ROS2的节点来调用雷达的SDK或者说API，来得到雷达的数据

得到了SDK的数据后，就可以进行后续的开发了，或者调用一个example

```bash
# 配置雷达节点


# 工作空间就按照这个来
mkdir -p ~/livox_ws/src
cd ~/livox_ws/src
git clone https://github.com/Livox-SDK/livox_ros_driver2.git
cd livox_ros_driver2




# 同样在当前目录，进入config文件夹
cd config
vim ./MID360_config.json # 不要MID360s,只要打开文件就行，不一定要求用vim

# 把host_net_info 中的192.168.1.5改成你自己的minipc的IP地址，保存退出就行了
# 我这里是192.168.1.150
#同时把lidar_net_info中的ip地址改成192.168.1.1xx，xx是SN的最后两位，SN码在雷达二维码上，自己看




# 找到build.sh，这个shell文件是这个节点的配置文件，配置完了才能在你自己的ubuntu上适配
./build.sh humble
# 第一个参数一定要选humble，可以自己阅读这个build.sh,如果参数选择ROS2,它会比选择humble少了一个配置





```

# 第三步，配置IP

在已经把环境配置好的情况下

最后一步，要具体考虑如何通信的问题，这里用的是网卡，那么就是局域网，显然是要配置IP地址的，雷达和minipc要在同一个局域网里，所以要把它们的IP地址配置成同一个网段的

打开局域网配置
IPv4 Method 改成手动
Address ：192.168.1.50 //这个也是上面那个.json文件里host_net_info的IP地址
Netmask：255.255.255.0

测试：
ping 192.168.1.1xx //xx是SN的最后两位

会显示64bytes 什么的，说明通信成功了


# 第四步，运行节点

```bash
# 运行节点
cd ~/livox_ws
source install/setup.bash
ros2 launch livox_ros_driver2 rviz_MID360_launch.py
```
