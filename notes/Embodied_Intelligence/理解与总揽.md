# 情景
假如我现在要做一个机械臂，但是我不可能直接做，因为在不清楚怎么做的情况下，很容易造成浪费

于是我们选择用仿真

# 心路历程
一切东西的制作，无非从 结构 - 硬件 - 软件 - 算法 - 应用

我们从结构入手，我们现在还没有开始实物操作，那么现在我们只有一些机械部件的stl文件，也就是常说的3d模型文件

有了这个我们就可以通过语言（python + MJCF）来设计一个机械臂

举个例子，stl文件是一些部件，而这个语言是一个工匠，工匠把这些部件拼接，形成一个机械臂
可以做到各种各样的机械臂

这个是仿真

仿真结束后，就可以实物制作一个机械臂，按照自己的设计

总而言之一句话，通过python + MJCF 的设计，可以模拟机械臂在各种情况下的情况

而只是把一个xml文件拖入mujoco，只能手动一些关节，不能模拟环境

# 总结
强调在不清楚实现路径时直接造机械臂容易浪费，因此应先用仿真；制作流程可拆为结构→硬件→软件→算法→应用。

目前只掌握机械部件的 STL 模型，通过 Python+MJCF 这套“工匠语言”把零件拼接为不同机械臂并在虚拟环境中测试，从而提前验证结构与控制策略。

仿真完成后可按同一设计制造实体机械臂，避免偏差；而仅把 XML 拖进 MuJoCo 只能手动拨动关节，不具备环境交互能力，远不如脚本化仿真。
