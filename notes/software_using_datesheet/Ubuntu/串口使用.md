# 使用分析

因为只打算在电脑上跑ros2

所以没有USB插，自然也没有/dev/ttyUSB0

那么此时就要下载一个虚拟串口

```bash
sudo apt intall socat

socat -d -d PTY,link=/tmp/ttyV0,raw,echo=0 PTY,link=/tmp/ttyV1,raw,echo=0

# 这条命令会创建两个虚拟设备：/tmp/ttyV0 和 /tmp/ttyV1


# 然后再输入这个命令就可以查看了
  minicom -D /tmp/ttyV0 -b 115200

```


但是串口出现bug了

```bash
Traceback (most recent call last):
  File "motor_init.py", line 16, in <module>
    serial = SerialPort('/tmp/ttyV0')
RuntimeError: IO Exception (25): Inappropriate ioctl for device, file /home/unitree/unitree_actuator_sdk-all/src/SerialPort.cpp, line 487.

```
说明你的“管理员”（Python 脚本）已经找到了库文件，并且成功开始执行了。但现在卡在了与硬件（或者虚拟硬件）打交道的阶段。

这个错误 RuntimeError: IO Exception (25): Inappropriate ioctl for device 是一个非常典型的底层通信错误。

具体到技术层面：
Unitree 的 SDK 为了追求高频率控制，会在底层调用一些只有真实串口驱动（比如 /dev/ttyUSB0 或 /dev/ttyS0）才支持的特殊设置（比如设置低延迟模式、特殊的比特率等）。
而你使用的 /tmp/ttyV0 显然是一个虚拟串口（通常是由 socat 或模拟器生成的），它不支持这些底层的硬件配置命令


解决：
1. 换真串口试试


2. 虽然报错是 ioctl，但有时候是因为权限不足导致无法正确读取设备属性。
sudo chmod 777 /tmp/ttyV0
